org.omg.CORBA public final class WrongTransaction extends UserException The CORBA WrongTransaction user-defined exception. This exception is thrown only by the methods Request.get_response and ORB.get_next_response when they are invoked from a transaction scope that is different from the one in which the client originally sent the request. See the OMG Transaction Service Specification for details. public WrongTransaction() Constructs a WrongTransaction object with an empty detail message. public WrongTransaction(String reason) Constructs a WrongTransaction object with the given detail message.
