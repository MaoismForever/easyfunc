javax.xml.xpath public abstract class XPathFactory extends Object An XPathFactory instance can be used to create XPath objects. See newInstance(String uri) for lookup mechanism. The XPathFactory class is not thread-safe. In other words, it is the application's responsibility to ensure that at most one thread is using a XPathFactory object at any given moment. Implementations are encouraged to mark methods as synchronized to protect themselves from broken clients. XPathFactory is not re-entrant.
