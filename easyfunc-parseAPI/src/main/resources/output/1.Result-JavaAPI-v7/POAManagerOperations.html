org.omg.PortableServer public interface POAManagerOperations Each POA object has an associated POAManager object. A POA manager may be associated with one or more POA objects. A POA manager encapsulates the processing state of the POAs it is associated with. void activate() throws AdapterInactive This operation changes the state of the POA manager to active, causing associated POAs to start processing requests. void hold_requests(boolean wait_for_completion) throws AdapterInactive This operation changes the state of the POA manager to holding, causing associated POAs to queue incoming requests. void discard_requests(boolean wait_for_completion) throws AdapterInactive This operation changes the state of the POA manager to discarding. This causes associated POAs to discard incoming requests. void deactivate(boolean etherealize_objects, boolean wait_for_completion) throws AdapterInactive This operation changes the state of the POA manager to inactive, causing associated POAs to reject the requests that have not begun executing as well as as any new requests. State get_state() This operation returns the state of the POA manager.
